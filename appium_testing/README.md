# Appium E2E Automation Testing Suite

This folder contains the complete, automated end-to-end (E2E) testing suite for the **Smart Food Waste Management** Android mobile application.

The tests are written in JavaScript (Node.js) using the **WebdriverIO** Appium client, and feature an automated reporting engine that outputs detailed Excel workbooks (`.xlsx`) with embedded status indicators and screenshot links.

---

## Folder Structure

```text
appium_testing/
├── .env                  # Configuration variables (ports, device name, APK path)
├── e2e_test.js           # Core automation script with the step-by-step workflow

├── package.json          # Node dependencies (webdriverio, exceljs, dotenv)

├── README.md             # This documentation file
├── reports/              # [AUTOGENERATED] Destination for Excel report output
└── screenshots/          # [AUTOGENERATED] Destination for step-by-step screenshots
```

## Scenarios Automated

The E2E test runs through a complete, realistic user interaction sequence:
1. **E2E_001: User Authentication**: Inputs valid credentials (`admin` / `1234`), clicks LOGIN, and verifies navigation to the Food Menu page.
2. **E2E_002: Food Menu Search**: Enters a search query ("Rice") to verify real-time search filtering.
3. **E2E_003: Add Food Item**: Opens the Add dialog, inputs a new food record (Day, Food Item, Quantity, Weather dropdown selection, Attendance, Switches), and submits.
4. **E2E_004: List Operations & Edit**: Increments the item quantity on the list card, opens the edit dialog, updates the quantity, and saves.
5. **E2E_005: Dashboard Analytics**: Clicks "Go To Dashboard", waits for the charts and calculated metrics (Total Sales, average demand, prediction, waste %) to render.
6. **E2E_006: AI Recommendations**: Clicks "View AI Recommendations" to load the final recommendation page showing forecasting suggestions.

## Configuration (.env)

Edit the `.env` file to customize your execution properties:
- `APPIUM_HOST`: Appium server host (default: `127.0.0.1`)
- `APPIUM_PORT`: Appium server port (default: `4723`)
- `DEVICE_NAME`: Name of your Android emulator or physical device (default: `Android Emulator`)
- `PLATFORM_VERSION`: Target Android operating system version (default: `12.0`)
- `APP_PATH`: Relative path to the compiled debug APK (default: `../build/app/outputs/flutter-apk/app-debug.apk`)

## How to Set Up & Execute

### Option A: The Fast Way (Windows Batch Runner)

We've provided a batch script that automates package installation, builds the latest version of the Flutter app, and runs the test suite.

Simply double-click `run_tests.bat` or run:
```powershell
.\run_tests.bat
```

### Option B: Step-by-Step Manual Run

#### 1. Prerequisites Set Up
Before running live automation, ensure:
1. **Android Studio & SDK** are configured.
2. **JAVA_HOME** and **ANDROID_HOME** environment variables are set in Windows.
3. An **Android Emulator** or device is running and detected:
   ```bash
   adb devices
   ```
4. **Appium CLI** is installed globally:
   ```bash
   npm install -g appium
   ```
5. **UIAutomator2 driver** is installed in Appium:
   ```bash
   appium driver install uiautomator2
   ```

#### 2. Run Appium Server
Open a separate command prompt and start Appium:
```bash
appium
```
*(By default, this runs on http://127.0.0.1:4723)*

#### 3. Compile the Flutter Debug APK
Navigate to the root Flutter project directory and compile:
```bash
flutter build apk --debug
```

#### 4. Run the Test Scripts
Navigate to the `appium_testing` directory, install packages, and execute:
```bash
npm install
node e2e_test.js
```

## Execution Modes

The automation suite is designed with two modes of execution to facilitate ease of review:

1. **Live Mode (Default)**:
   - Activates when Appium is detected on port `4723` and a valid APK is found.
   - Installs the app, opens the emulator, performs actual E2E taps and inputs, takes active screenshots, and saves the Excel report.
   - You can force live execution to fail/warn if Appium is down by running: `node e2e_test.js --live`

2. **Simulated Mode (Fallback)**:
   - Automatically activates if the Appium server is offline or the APK is not built.
   - Generates simulated steps, writes valid 1x1 pixel screenshot placeholders, and outputs the identically formatted Excel report.
   - This allows you to **instantly verify the reporting engine, files creation, and formatting** without starting an emulator or Appium.

## Excel Test Analysis Design

The Excel file generated under `reports/` is styled to professional criteria:
- **Sheet 1: Execution Summary**:
  - Highlights title, metrics (total, passed, failed), pass rate, durations, and environment details.
  - Features the custom Orange theme matching the Smart Food Waste application.
- **Sheet 2: Detailed Log**:
  - Detailed rows showing Scenario, Action, Expected, Actual, Status, and Elapsed Time.
  - **Color-Coded Statuses**: Safe soft-green fills for `PASS` and soft-red fills for `FAIL`.
  - **Clickable Hyperlinks**: The screenshot column contains clickable hyperlinks (`View Screenshot 📸`) that open the exact step screenshot stored inside the `screenshots/` directory directly from your Excel viewer.
